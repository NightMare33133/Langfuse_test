## 核心原则

**每条查询必须满足：**
1. **单一概念** — 只对应一个明确的、可在当前输入文本中定位的原文事实或定义
2. **单证据片段** — 金标准原文证据可从一个连续的原文片段中完整找到
3. **无需聚合** — 不需要从多个分散位置拼凑信息
4. **无需推理** — 金标准证据是原文的直接陈述，不需要归纳、推导或对比
5. **可为近义改写** — 查询可以与原文措辞不同，但只能是单一概念的近义/语义改写，用于测试 embedding 检索；不得引入文中不存在的信息

---

## query（检索查询）规范（重要）

**query 必须是短检索查询，可为词、词组、短语或单一检索意图：**
- ✅ 可以是：名词、名词短语、动宾短语、"修饰语+核心概念"
- ✅ 可以与原文措辞不同（近义改写），但只能是单一概念
- ✅ **优先采用同义词、语序调整或等价表述**，以测试 embedding 的语义检索能力
- ❌ **不得是问句** — 禁止出现问号（？）
- ❌ **禁止问答或推理导向表达** — 不得包含"什么/为何/为什么/如何/是否/哪些/请分析/请说明/分别"等词
- ❌ **禁止多子问题** — 不得包含比较、归因、总结、影响分析、步骤分析等需要模型组织回答的内容
- ❌ **不得引入文中不存在的信息**
- ❌ **不得直接逐字照抄 reference_answer 中的连续核心短语** — 专有名词、条款编号或文中定义术语除外；应通过语义改写生成查询

---

## reference_answer（金标准原文证据）规范（重要）

**reference_answer 必须从当前知识库片段中逐字复制一个完整、连续、可独立表达的原文证据单元：**
- 通常应为完整段落；
- 若原文定义、金额、期限或单点事实本身只有一个完整句子，允许使用该完整句子；
- 逐字复制，不得翻译、概括、改写、拼接不同位置或编造
- **不要扩写** — 不要把原文的一句话扩展成一段分析
- **不要添加** — 不要添加原文中没有的解释、背景、补充说明
- 禁止截取半句或拼接不同位置的内容
- 不要使用或输出 Python 的 chunk_id 作为金标准，因为 Dify 的实际分块边界可能不同
- 如果 chunk 中无法识别出足以独立表达证据的完整证据单元，则跳过该样本

**示例：**
```
原文片段："RAG（Retrieval-Augmented Generation）是一种结合信息检索与文本生成的技术框架。"
检索查询：RAG 技术框架定义
✅ 正确金标准证据："RAG（Retrieval-Augmented Generation）是一种结合信息检索与文本生成的技术框架。"
❌ 错误金标准证据："RAG，全称 Retrieval-Augmented Generation，是一种先进的技术框架，它巧妙地将信息检索与文本生成相结合..."（扩写了）
```

## 查询类型优先级

### 优先生成（最纯粹的单跳检索查询）

**定义类**
- "XXX 定义"
- "XXX 概念"
- "XXX 含义"

**枚举/列表类**
- "XXX 类型/类别"
- "XXX 组成部分"
- "XXX 步骤/阶段"

**单点事实类**
- "XXX 数值/比例"
- "XXX 时间/日期"
- "XXX 负责人/提出者"
- "XXX 公式/表达式"

### 条件生成（仅在证据明确时）

**特点/原理类** — 仅当原文用一两句话明确概括了特点或原理时
- "XXX 工作原理"
- "XXX 特点"

**判断条件**：金标准证据必须是原文中的一段连续描述，而不是需要你从多处归纳的。

---

## 明确禁止的查询类型

### ❌ 对比/区别类
- "A 和 B 的区别"
- "A 与 B 相比优势"
**原因**：需要找 A、B 两处信息再归纳，属于多跳

### ❌ 优缺点分析类
- "XXX 优缺点"
- "XXX 局限性"
**原因**：通常需要从多处收集信息

### ❌ 原因分析类
- "为什么 XXX"
- "XXX 原因"
**原因**：因果关系需要推理或多证据聚合

### ❌ 影响/意义类
- "XXX 影响"
- "XXX 重要意义"
**原因**：需要综合分析，证据不唯一

### ❌ 开放式/问答类
- "如何理解 XXX"
- "XXX 是什么"
- "请分析 XXX"
**原因**：问答句式，不是检索查询

## 输出格式

请严格输出一个 JSON 数组，不要包含任何其他文字。每个元素包含以下字段：

```json
[
  {{
    "question": "短检索查询（词/词组/短语，非问句）",
    "reference_answer": "从当前片段逐字复制的金标准原文证据",
    "source_excerpt": "与 reference_answer 完全相同",
    "difficulty": "事实/基础",

  }}

```

**注意**：`source_excerpt` 字段必须与 `reference_answer` 完全一致，不得截断、改写或扩写。

## 自检清单（每条查询必须通过）

- [ ] **非问句** — query 中不含问号、不含"什么/为何/为什么/如何/是否/哪些/请分析/请说明/分别"等问答导向词？
- [ ] **语义改写** — query 是否采用了同义词/语序调整/等价表述，而非逐字照抄证据中的连续核心短语？
- [ ] **单一概念** — 是否只对应一个知识点？（不是对比两个概念）
- [ ] **完整证据** — reference_answer 是否是一个完整、连续、可独立表达的原文证据单元（完整段落或完整句子）？
- [ ] **逐字复制** — reference_answer 是否是原文的逐字复制？（没有扩写、概括、改写）
- [ ] **source_excerpt = reference_answer** — 两个字段是否完全一致？
- [ ] **可命中** — 如果 RAG 检索到该证据片段，能否直接支撑该查询？

**任何一项不通过，请跳过该查询，不要凑数。**
